# Writing Plans

## When to use this skill

- Before starting any integration or architecture change.
- When a task takes more than 5-10 small steps.

## Workflow

1. State the Goal clearly.
2. Research dependencies and existing logic.
3. Define User Review Required sections for major decisions.
4. List Proposed Changes file by file.
5. Define a Verification Plan (Automated + Manual).

## Instructions

- Keep plans modular and logical.
- Stop for user review on all high-impact changes.
