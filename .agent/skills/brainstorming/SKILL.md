# Brainstorming

## When to use this skill

- Before starting any new feature or layout.
- When a requirement is ambiguous.

## Workflow

1. Explore project context (files, docs).
2. Propose Visual Companion for UI tasks.
3. Ask clarifying questions one-by-one.
4. Propose 2-3 implementation options.
5. Get user feedback on segments.
6. Write a design spec.

## Instructions

- DO NOT start coding until the design is approved.
- Use writing-plans skill after approval.
