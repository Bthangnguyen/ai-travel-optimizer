# Test-Driven Development (TDD)

## When to use this skill

- When building new validation logic (Parser).
- When implementing complex mathematical algorithms (Routing Solver).

## Workflow

1. Write a failing test for the desired feature.
2. Run the test to confirm it fails.
3. Write the minimum code necessary to make the test pass.
4. Run all tests to ensure no regressions.
5. Refactor the code for better quality.

## Instructions

- Use pytest or unittest as per project standards.
- Focus on edge cases (e.g., negative time windows, missing POI coordinates).
